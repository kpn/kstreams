    kstreams.ProducerSettings

    Bases: BaseModel

    Kafka producer configuration.

    Attributes:

    Name Type Description
    client_id Optional[str]

    Name passed to Kafka brokers for request logging. Default to None.
    metadata_max_age_ms int

    Interval in milliseconds to force metadata refresh. Default to 300000 (5 minutes)
    request_timeout_ms int

    Produce request timeout in milliseconds. Default to 40000 (40 seconds)
    acks Union[Literal[0, 1], Literal['all']]

    Required broker acknowledgments for produce requests. Default to 1.
    compression_type Optional[Literal['gzip', 'snappy', 'lz4', 'zstd']]

    Compression algorithm used for produced batches. Default to None
    max_batch_size int

    Maximum buffered bytes per partition before send blocks. Default to 16384
    partitioner Optional[Callable[..., int]]

    Callable used to choose target partition. Default to None.
    max_request_size int

    Maximum size of a produce request in bytes. Default to 1048576 (~1 MB)
    linger_ms int

    Time in milliseconds to wait for additional records before send. Default to 0
    retry_backoff_ms int

    Backoff in milliseconds between retry attempts. Default to 100
    connections_max_idle_ms Optional[int]

    Close idle connections after this timeout. Default to 540000 (9 minutes)
    enable_idempotence bool

    Enables idempotent production semantics. The value of acks must be all when enable_idempotence is True. Default to False.
    transactional_id Optional[str]

    Transactional producer id for transactions. Default to None
    transaction_timeout_ms int

    Transaction timeout in milliseconds. Default to 60000 (1 minute)

    Raises:

    Type Description
    ValueError

    If enable_idempotence is True and acks is 0 or 1.

    Example

    from kstreams import create_engine
from kstreams.backends.kafka import Kafka
from kstreams.clients import ProducerSettings


backend = Kafka(bootstrap_servers=["localhost:9092"])
producer_settings = ProducerSettings(
    client_id="my-producer",
    linger_ms=10,
    request_timeout_ms=80000,
)

stream_engine = create_engine(
    backend=backend,
    producer_settings=producer_settings
)

    kstreams.ConsumerSettings

    Bases: BaseModel

    Kafka consumer configuration.

    Attributes:

    Name Type Description
    group_id str

    Name of the consumer group to join.
    client_id Optional[str]

    Name passed to Kafka brokers for request logging. Default to None.
    group_instance_id Optional[str]

    Unique identifier for static consumer group membership. Default to None
    fetch_max_wait_ms int

    Maximum time in milliseconds the broker waits to fill fetch requests. Default to 500 (0.5 seconds)
    fetch_max_bytes int

    Maximum bytes to fetch in a single request. Default to 52428800
    fetch_min_bytes int

    Minimum bytes to fetch in a single request. Default to 1
    max_partition_fetch_bytes int

    Maximum bytes to fetch per partition in a single request. Default to 1048576 (~1 MB)
    request_timeout_ms int

    Fetch request timeout in milliseconds. Default to 40000
    retry_backoff_ms int

    Backoff in milliseconds between retry attempts. Default to 100
    auto_offset_reset Literal['latest', 'earliest', 'none']

    What to do when there is no initial offset or the current offset is invalid. Default to latest
    enable_auto_commit bool

    Whether to automatically commit offsets. Default to True
    auto_commit_interval_ms int

    Interval in milliseconds to automatically commit offsets. Default to 5000 (5 seconds)
    check_crcs bool

    Whether to check CRCs on messages consumed from Kafka. Default to True
    metadata_max_age_ms int

    Interval in milliseconds to force metadata refresh. Default to 300000 (5 minutes)
    partition_assignment_strategy Tuple[Type, ...]

    List of partition assignor classes to use for consumer group coordination. Default to RoundRobinPartitionAssignor
    max_poll_interval_ms int

    Maximum time in milliseconds between calls to poll() before the consumer is considered failed. Default to 300000 (5 minutes)
    rebalance_timeout_ms Optional[int]

    Time in milliseconds to wait for a rebalance to complete before raising an error. We decouple this setting to allow finer tuning by users that use RebalanceListener to delay rebalacing If not set, defaults to session_timeout_ms.
    session_timeout_ms int

    Timeout in milliseconds for consumer group session. Default to 10000 (10 seconds)
    heartbeat_interval_ms int

    Interval in milliseconds to send heartbeats to the consumer group coordinator. Default to 3000 (3 seconds)
    consumer_timeout_ms int

    Time in milliseconds to wait for messages in poll() before returning an empty result. Default to 200 (0.2 seconds)
    max_poll_records Optional[int]

    Maximum number of records returned in a single call to poll(). Default to None (no limit)
    exclude_internal_topics bool

    Whether to exclude internal topics when auto-subscribing to topics by regex. Default to True
    connections_max_idle_ms Optional[int]

    Close idle connections after this timeout. Default to None (no timeout)
    isolation_level Literal['read_uncommitted', 'read_committed']

    Controls how to read messages written by transactional producers. Default to read_uncommitted

    Example

    from kstreams import create_engine, ConsumerSettings, ConsumerRecord

consumer_settings = ConsumerSettings(
    group_id="example-group",
    enable_auto_commit=False,
    auto_offset_reset="earliest",
)

stream_engine = create_engine(title="my-stream-engine")


@stream_engine.stream("my-topic", settings=consumer_settings)
async def stream(cr: ConsumerRecord) -> None:
    print(f"Event consumed: headers: {cr.headers}, payload: {cr.value}")
